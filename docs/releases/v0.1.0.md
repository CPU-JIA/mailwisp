# MailWisp v0.1.0

首个独立产品Release。MailWisp提供Canonical匿名Inbox API、Vue Web Console、Postfix→LMTP可靠收件、PostgreSQL事实状态、Content-addressed Raw MIME存储、DuckMail/YYDS/Cloudflare Temp Email可选Adapter，以及Canonical Docker Compose部署、备份恢复和可观测性基线。

## Breaking Change

- 无。该版本是MailWisp首个Release，不存在需要兼容的旧MailWisp正式版本。
- MailWisp不是旧TempMail的升级版，不读取、不迁移也不兼容旧产品数据、Token或配置。

## Migration与数据影响

- 仅支持全新空数据库安装。
- 首次启动由`migrate`服务应用MailWisp自身不可变Migration。
- 生产升级前必须创建并独立验证一致性Backup Bundle；不得把已前向迁移的数据库直接交给旧版本。

## 配置变化

- Canonical部署使用`deploy/compose/`，只公开`25/tcp`、`80/tcp`和`443/tcp`。
- 必须分别提供PostgreSQL Password、Browser Session Key和Create Quota HMAC Key，三者不得复用。
- 必须配置Web Domain、SMTP Host、Mail Domain、Certificate Name与Public Domains。
- DuckMail、YYDS和Cloudflare Temp Email Adapter默认关闭，按需显式启用。

## 已验证范围

- Go Test、Race、Vet、Fuzz、Govulncheck、Gosec、Gitleaks与PostgreSQL/Postfix Integration。
- Vue Type Check、Lint、Unit、Browser E2E、Production Build与npm audit。
- Canonical Compose HTTPS/SMTP/Postfix/LMTP/Parser/Browser E2E、容量Benchmark与删除源Volume后的灾备恢复。
- Linux amd64 Release使用固定Buildx与Digest BuildKit执行两次隔离无缓存构建并证明Archive逐字节一致；同时提供SPDX 2.3 SBOM、Final Image/IaC扫描、Checksum、解压加载镜像后的Production E2E，以及Release Subject与Checksum Manifest的GitHub Artifact Attestation。
- Candidate Artifact与Draft Release使用同一扁平发布目录和Basename Checksum，下载后无需重建目录即可完整校验；Source SBOM只扫描与Build Commit绑定的隔离`git archive`快照。

公网DNS、MX、真实ACME、云厂商25端口、PTR、外部SMTP信誉、宿主机断电和云盘硬故障仍必须在目标基础设施独立验收。

## Rollback

1. 保留旧Compose配置、旧镜像和升级前已验证Backup，禁止删除旧Volume。
2. 如果没有Schema前向变化，可在隔离Project使用旧镜像验证后切换流量。
3. 如果已经应用不兼容Migration，必须恢复到新的空PostgreSQL与Content Volume，不能对当前Volume原地降级。
4. 回滚后重新验证HTTPS、SMTP STARTTLS、Postfix Queue、Browser Session、附件读取和新邮件投递。
